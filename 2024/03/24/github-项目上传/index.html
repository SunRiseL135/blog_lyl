github-项目上传
发表于 2024-03-24 | 更新于 2024-03-24
github
教程

通过命令行上传项目到github中，了解其执行原理…

参考：https://blog.csdn.net/dwz100916007/article/details/101271556

步骤：

  1. Create a new repository

    按照需求填写仓库内容，然后创建，就可以在仓库列表看到新建的仓库信息了。

    我们需要注意该仓库的 https 链接网址，这是你本地链接远程仓库的网址，也可以用 ssh。

  2. 打开命令终端,进入到你要提交代码的项目的文件夹

  3. 本地初始化一个空Git仓库,相当于在当前项目的目录中生成本地的git管理

  4. 输入git status查看当前的状态

  5. 将项目上所有的文件添加到仓库中

  6. 把项目提交到仓库中去

    -m 后面的是提交的注释,最好填一下,不填容易报错

  7. 把本地仓库关联到 github 上去（这是就用到了上文中的 https 的网址）

  8. 最后一步,把代码push到仓库

    可能出现的错误如下，

    这是因为在创建仓库的时候点选了默认创建 readme 文件,所以先要把两地文件进行合并

    合并之后再重新git push -u origin master ，然后刷新 github 的仓库,则会看到代码已经上传。
